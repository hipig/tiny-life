# 交互规则

## 主交互模型

公寓楼是主要 UI。

玩家主要通过以下方式操作：

- 单指拖动移动公寓视图。
- 双指缩放观察公寓。
- 点击房间打开房间 UI。
- 点击公共区、服务核心/电梯厅或屋顶打开对应装修面板。
- 点击租客打开租客状态。
- 长按家具移动或回收。
- 点击可建造房间进行建造。

## 缩放层级

概念上支持三个观察层级：

- 远景：查看楼层、建造槽、整体公寓状态。
- 中景：日常经营，点击房间、查看家具和租客。
- 近景：观察小人行为、气泡、家具互动。

## 点击优先级

普通状态下输入优先级从高到低：

1. 弹窗 UI。
2. 顶部状态栏。
3. 右侧悬浮按钮。
4. 租客。
5. 家具长按。
6. 房间气泡或金币提示。
7. 公共区、服务核心/电梯厅或屋顶装修目标。
8. 房间空白区域。
9. 可建造房间。
10. 拖动画面。

## UI 状态机

通过 `UIManager` 维护唯一 UI 状态来源。

预期状态：

```gdscript
enum UIState {
    NORMAL,
    ROOM_PANEL,
    SPACE_DECOR_PANEL,
    FURNITURE_SHOP,
    PLACING_NEW_FURNITURE,
    MOVING_EXISTING_FURNITURE,
    TENANT_PANEL,
    BUILD_CONFIRM,
    TASK_PANEL,
    APARTMENT_OVERVIEW,
    INCOME_DETAIL,
    RENT_DETAIL,
    REWARD_PANEL,
    SETTINGS_PANEL,
    POPUP
}
```

## 状态权限

`NORMAL` 允许：

- 点击房间。
- 点击公共区、服务核心/电梯厅和屋顶装修目标。
- 点击租客。
- 长按家具。
- 点击可建造房间。
- 拖动/缩放相机。

`ROOM_PANEL` 允许：

- 切换房间 UI Tab。
- 在装饰 Tab 中切换房间装修。
- 打开家具商店。
- 打开租客招募。
- 可选支持点击其他房间切换目标房间。

`SPACE_DECOR_PANEL` 允许：

- 切换当前目标支持的装修分类。
- 选择装修条目。
- 关闭装修面板。
- 可选支持点击其他装修目标切换目标。

`FURNITURE_SHOP` 允许：

- 选择家具。
- 关闭商店。

`PLACING_NEW_FURNITURE` 允许：

- 拖动家具预览。
- 可选平移/缩放视图。
- 确认摆放。
- 取消摆放。

`PLACING_NEW_FURNITURE` 禁止：

- 打开房间 UI。
- 点击租客。
- 点击建造。
- 长按其他家具。

`MOVING_EXISTING_FURNITURE` 允许：

- 拖动当前家具。
- 确认位置。
- 取消移动。
- 回收家具。

## 顶部状态栏

顶部状态栏只显示：

- 公寓等级。
- 金币/收益。
- 租金。

点击行为：

- 点击公寓等级打开公寓总览。
- 点击金币/收益打开收益详情。
- 点击租金打开租金构成详情。

金币增长提示：

- 不要每秒显示。
- 自动收益应合并后约每 5 到 10 秒显示一次。
- 家具摆放、招募租客、升级、离线收益领取后可以显示更明显反馈。

## 房间 UI

点击房间空白区域打开房间 UI。

推荐 Tab：

- 概览。
- 家具。
- 租客。
- 装饰。

MVP 可以先做：

- 家具。
- 租客。

家具商店只能从目标房间的家具 Tab 打开。

租客招募只能从目标房间的租客 Tab 打开。

装修系统按场景目标进入：房间从 RoomPanel 的装饰 Tab 打开；公共区、服务核心/电梯厅和屋顶从对应场景目标直接点击打开 `SpaceDecorPanel`。不新增底部导航或全局装修商城入口。

装修分类、价格、预览和主题来自 `room_decor.json`。房间支持墙纸、墙体、房门；公共区支持墙纸、墙体，存在入口门时支持房门；服务核心/电梯厅支持墙纸、墙体；屋顶只支持屋顶样式。电梯门本体本期不参与换装。

租客招募区域、候选数量和租金承受上限来自 `tenant_regions.json`，不得在 TenantPanel 中补默认区域或默认文案。

## 家具规则

不存在家具仓库。

家具只存在两种状态：

- 配置表中的可购买家具。
- 房间中已摆放的家具实例。

新家具流程：

```text
在房间绑定的商店中选择家具
进入摆放预览
拖动并按连续坐标调整位置，地面家具保持地面线对齐
确认合法位置
扣除金币
创建家具实例
刷新房间评分和租金
存档
```

取消摆放不得扣金币。

MVP 不支持旋转。

后续最多考虑左右镜像。

已有家具：

- 短按不打开详情。
- 长按进入家具操作状态。
- 移动不扣金币。
- 取消移动恢复原位置。
- 回收删除实例，并按配置返还金币，默认返还 50%。

摆放合法性必须检查：

- 不超出目标房间。
- 不与已有家具重叠。
- 不挡门。
- 墙面家具只能放墙面区域。
- 地面家具只能放地面区域。
- 需要靠墙的家具必须靠墙。

## 建造规则

禁止添加全局建造按钮。

建造发生在公寓场景内，通过待建房间槽触发。

建造槽状态：

```gdscript
enum RoomBuildSlotState {
    HIDDEN,
    BUILDABLE,
    COMPLETED
}
```

可建造条件：

- 房间尚未建成。
- 公寓等级达到房间配置要求。
- 更低楼层的出租房间已全部建成；同一层左右房间互不锁顺序。

金币数量不影响待建房间是否显示。金币只影响确认按钮是否可点击。
